A Bioinformatics Pipeline for Biologically Relevant HVG Selection
in scRNA-seq Data for Training Predictive DL Models

One-line Summary

  • A flexible and biologically informed pipeline that refines highly variable genes from single-cell RNA-seq data to optimize deep learning model training.

1. Introduction

  • Standard HVG selection methods often overlook biological context, leading to noisy or irrelevant gene sets for model training.
  • We developed a biologically grounded pipeline that benchmarks HVGs against immune-relevant genes to refine feature sets.
  • Originally designed for kidney cancer + ICB therapy, but customizable for other contexts.

2. Pipeline Overview

  • Step 1: Extract HVGs from selected cells
  • Step 2: Evaluate biological relevance of HVG sets
  • Step 3: Intersect optimal HVG set with known relevant genes

3. Step-by-Step Breakdown

3.1 Generate HVG Sets

  • Start with 10X scRNA-seq files.
  • Select relevant cells (e.g. PR, SD, PD), save in a new directory.
  • Use Seurat to compute HVGs incrementally (100 to 9,000).

Figure 3.1. Distribution of HVGs per set. (To be added)

3.2 Assess Biological Relevance

  • Benchmark HVG sets against curated gene lists (GO terms, pathways).
  • Use biomaRt and msigdbr to pull immune-relevant genes.
  • Plot overlaps and identify the optimal cutoff (elbow point).

Figure 3.2. Overlap curve of HVGs vs. relevant gene list. (To be added)

3.3 Extract Refined HVG Set

  • Intersect optimal HVG set with biologically relevant genes.
  • Result is a robust gene set for model input.
