A Bioinformatics Pipeline for Biologically Relevant HVG Selection in scRNA-seq Data for Training Predictive DL Models

A flexible and biologically informed pipeline that refines highly variable genes from single-cell RNA-seq data to optimize deep learning model training.

1. Introduction

  • In high-dimensional single-cell transcriptomic datasets, selecting meaningful input features for machine learning or deep learning (DL) models poses a significant challenge.
  • Standard approaches typically rely on selecting the top 2,000 to 5,000 highly variable genes (HVGs) using purely statistical criteria. However, many of these genes may be noisy or unrelated to the specific biological context, such as the disease type or therapy being studied.
  • We developed a bioinformatics pipeline tailored for single-cell RNA-seq (scRNA-seq) data in 10X format. This pipeline integrates statistical variability with biological relevance by benchmarking HVG sets against a curated list of genes known to be involved in context-specific pathways.
  • This biologically anchored approach increases the interpretability and accuracy of predictive models.
  • While the initial use case involved kidney cancer patients treated with immune checkpoint blockade (ICB) therapy, the pipeline is designed to be generalizable.

2. Pipeline Overview

  • Step 1: Extract HVGs from selected cells
  • Step 2: Determine the optimal HVG set by evaluating biological relevance
  • Step 3: Generate a refined gene list by intersecting the HVG set with biologically relevant genes

3. Step-by-Step Breakdown

3.1 Step One: Generate HVG Sets

  • The pipeline starts with scRNA-seq input files in 10X format.
  • Users can specify which cell populations to include ‚Äî for instance, based on therapy response categories such as partial response (PR), stable disease (SD), or progressive disease (PD).
  • Using Seurat, the pipeline calculates HVGs and saves them incrementally in sets of 100, from 100 up to 9,000.
  • These sets represent candidate feature spaces for downstream evaluation.
Figure 3.1. Distribution of HVGs in each incremental set. (To be added later.)

3.2 Step Two: Assess Biological Relevance of HVG Sets

  • Each HVG set is benchmarked against a reference list of biologically meaningful genes.
  • These genes are extracted using biomaRt and msigdbr based on curated GO terms and pathways from literature.
  • The overlap is quantified to measure the interpretability of each HVG set.
  • The goal is to find the inflection threshold where adding more HVGs no longer increases biological relevance.
Figure 3.2. Curve showing overlap between HVG sets and immune-related genes. (To be added later.)

3.3 Step Three: Extract Biologically Relevant HVGs for Model Training

  • Intersect the optimal HVG set with the reference gene list to extract the final list of biologically informed HVGs.
  • This refined gene list can then be used as input for deep learning or machine learning models.
Figure 3.3. Final refined gene list visualization. (To be added later.)

4. Output and Results

  • The output is a refined CSV file of HVGs optimized for biological relevance.
  • Figures include HVG distributions, overlap curves, and inflection point visuals.
  • The modular pipeline supports disease-specific customization through interchangeable GO terms and pathways.
  • Initially tested on scRNA-seq data from kidney cancer patients undergoing ICB therapy, but applicable to other disease areas.
